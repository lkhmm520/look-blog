本文将手把手教你如何使用在线工具，加载我维护的**个人自用配置（CustomRules）**，并带你**看懂**转换后的策略组，彻底告别“节点连不上”的困惑。

## 🚀 基础教程：一键转换

### 第一步：填写基础信息

1.  打开 [ACL4SSR](https://acl4ssr-sub.github.io/)。
2.  在 **“订阅链接”** 中粘贴你的机场订阅地址（支持多个链接用 `|` 分隔，或直接换行）。
3.  **“后端地址”** 建议选择列表中的 **自建后端** 或 `api.wcc.best` 等高可用后端，以确保转换速度和稳定性。

### 第二步：加载自定义规则（关键）

在 **“远程配置”** 下拉框中，选择 **“Custom（自定义）”**，然后将上面准备好的 **个人自用 规则地址** 粘贴进去。

{{< figure src="step_config.png" caption="在远程配置中填入自定义 URL" class="rounded-lg shadow-md" >}}

### 第三步：生成并导入

1.  **客户端**：选择你使用的软件（如 Clash、Surge 等）。
2.  点击 **“生成订阅链接”**。
3.  点击 **“一键导入”** 即可在客户端中看到新的配置文件。

## 🎨 进阶技巧：利用网页功能微调节点

如果你不想 fork 代码，只是想简单地**过滤掉不需要的节点**（比如“过期时间”、“官网”等无用节点），或者**只保留特定国家**的节点，可以直接利用 ACL4SSR 网页自带的高级功能。

点击页面上的 **“更多选项”** 展开高级菜单：

### 1. 排除节点 (Exclude Nodes) - 正则表达式
有些机场订阅会包含很多无效信息节点，我们可以通过关键词把它们过滤掉。
* **操作**：在“排除节点”输入框中填写正则表达式。
* **示例**：
    ```regex
    (流量|时间|官网|产品|过期|剩余)
    ```
    *解释：名字里只要包含“流量”、“官网”等字眼的节点，统统不要。*

### 2. 包含节点 (Include Nodes) - 只留精华
如果你只想要 AI 友好的节点（如美国、日本、新加坡），可以使用此功能。
* **操作**：在“包含节点”输入框中填写。
* **示例**：
    ```regex
    (US|SG|JP|美国|新加坡|日本)
    ```
    *解释：转换出来的订阅将**只包含**这些国家的节点，其他统统丢弃。*

### 3. 重命名 (Rename) - 强迫症福音
如果机场节点的命名很乱，你可以通过“正则替换”功能来批量修改。
* 举例：把 `[高级机场] 香港 01` 改成 `🇭🇰 香港 01`，让列表更整齐。

## 🧐 策略组全解：转换后怎么选？

很多朋友导入后，看到一堆“自动选择”、“手动选择”、“漏网之鱼”就懵了。这里详细解释一下我的规则逻辑，助你根据场景精准选择。

### 1. 🚀 节点选择 (Proxy)
这是**总开关**。
* **推荐设置**：通常选 `自动选择` 或 `手动选择`。
* **作用**：决定所有需要走代理的流量默认走哪条路。

### 2. 🤖 自动选择 (Auto / UrlTest)
* **机制**：客户端会自动测试所有节点的延迟（Ping），并自动切换到**延迟最低**的那个节点。
* **适用场景**：懒人首选，不需要关心连哪个节点，系统自动帮你选最快的。

### 3. 🌍 漏网之鱼 (Final / Match)
* **含义**：当一个网址**既不在**国内直连名单，**也不在**广告屏蔽名单，且**没有被任何规则命中**时，它就会落到这里。
* **推荐设置**：建议设置为 `节点选择` 或 `自动选择`。
* **注意**：如果你发现某些冷门国外网站打不开，检查一下这个选项是不是被设成了 `Direct（直连）`。

### 4. 🧠 AI 平台 (ChatGPT/Gemini/Claude)
这是最重要的设置！因为 OpenAI 等平台对地区限制极严。
* **❌ 避坑指南**：**千万不要选香港 (HK)、澳门 (MO) 或台湾 (TW) 节点**，否则极大概率会被封号或无法访问。
* **✅ 推荐设置**：请手动指定 **新加坡 (SG)**、**日本 (JP)** 或 **美国 (US)** 节点。

### 5. 📺 流媒体/应用分组 (Netflix/Telegram 等)
我的规则对常见服务做了独立分组。
* **作用**：你可以为 Netflix 单独指定一个“新加坡”节点，同时让 Telegram 走“美国”节点，互不干扰。
* **推荐设置**：`自动选择` 或手动指定对应地区的节点。

### 6. 🛑 广告拦截 (AdBlock)
* **推荐设置**：`REJECT`（拒绝连接）。
* **作用**：直接阻断广告域名的请求，节省流量并净化页面。

## 🛠️ 开发者玩法：完全掌控规则

如果你是一名开发者（像我一样是前端开发），你可能会对现有的规则仍不满意，想要自己定义策略组。

### 关于维护规则
`CustomRules.ini` 本质上是一个模板文件。在这个文件中，你可以定义：
* **Rule Sets（规则集）**：引用外部的 list 文件（如 GeoIP、AdBlock 列表）。
* **Proxy Groups（策略组）**：定义“自动选择”、“手动选择”、“故障转移”等分组逻辑。

如果你想修改我的规则，可以 Fork 我的 GitHub 仓库 [`WebTeen/ssr-rules`](https://github.com/WebTeen/ssr-rules)，修改 `CustomRules.ini` 后，在转换时填入你自己的 Raw URL 即可。
